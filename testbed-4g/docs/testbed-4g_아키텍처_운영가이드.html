testbed-4g — 아키텍처 & 운영 가이드

multi-eNB 최종형 · Open5GS EPC + srsRAN(ZMQ) + ArduPilot MAVLink C2 · 전 게이트 실측 통과(2026-07-01)
구현 로컬 Docker (WSL2 + Docker Desktop) 근거 Spiderweb + Sonaglio et al. arXiv:2603.04662 멀티UE multi-eNB (브로커 회피) 역할경계 인프라/배선만 — 공격은 사용자

1. 최종 아키텍처 (multi-eNB)

(eNB+UE) 쌍을 UE 수만큼 띄워 멀티UE를 구현(GNU Radio 브로커 회피). 3 쌍 모두 같은 EPC에 S1AP 연결 → 같은 APN 풀(192.168.100.0/24)에서 UE-to-UE 도달(PGW hairpin) → C2·TM1 성립. GPS는 dahnet(비셀룰러)로 분리 공급.

APP (SITL/GCS/주입기) UE / C2 (셀룰러, 192.168.100/24) RAN (eNB, ZMQ) EPC 코어 (172.22/16) GPS (dahnet, 172.28/16, 비셀룰러) ROGUE (TM1)
multi-eNB 최종형 — (eNB+UE)×3 → 단일 EPC · C2=셀룰러 · GPS=dahnet 분리
왜 multi-eNB인가. srsRAN ZMQ는 1 eNB=1 UE(가상 RF 파이프 1:1). 1 eNB에 N UE를 붙이려면 GNU Radio 브로커(샘플레이트·타이밍 정합 = 최대 난점)가 필요. 대신 eNB를 UE 수만큼 띄우면 각 쌍이 검증된 단일 UE 구성이라 안정적이고, 모두 같은 EPC에 붙으므로 UE-to-UE(C2/TM1)는 그대로 성립한다.

5. 동작 과정 (환경 → 폐루프)

  1. 호스트 준비 — WSL2 공유커널에 sctp 로드(MME S1AP 전제), /dev/net/tun 확인. (00-local-prep)
  2. EPC 기동 — docker_open5gs 빌드 → 코어 9서비스 --no-deps 기동 → MME가 S1AP(:36412)·S6a(HSS) 개방. (10-epc-up)
  3. 가입자 등록 — UE의 IMSI/Ki/OPc를 HSS에 등록(OPc는 OP+Ki로 계산). 미일치면 attach 거부. (provision · 30-add-ue)
  4. UE attach ×3 — 각 srsue가 ZMQ로 자기 eNB에 동기(RACH) → eNB가 MME에 S1 Setup → 인증 → 기본 베어러 → tun_srsue IP(192.168.100.x).
  5. UE-to-UE — 같은 APN 풀이라 UPF hairpin으로 UE끼리 직접 IP 도달(C2·TM1 전제). ICMP+UDP14550 검증.
  6. C2 over LTE — SITL(UAV-UE netns)이 MAVLink를 tun_srsue로 GCS-UE에 송출 → 셀룰러 경유 양방향(HEARTBEAT↓/COMMAND_ACK↑).
  7. GPS off-셀룰러 — SITL을 외부GPS(SIM_GPS_DISABLE=1/GPS_TYPE=14)로 전환, gps_inject가 dahnet으로 GPS_INPUT @10Hz 공급 → fix_type=3.
  8. 폐루프 비행 — GCS가 셀룰러 C2로 GUIDED→arm→takeoff→land. GPS=dahnet·C2=셀룰러 동시 정상.

6. 기능별 사용법

bootstrap.sh 환경→G0

wsl -d Ubuntu cd .../testbed-4g bash bootstrap.sh # 00-local-prep → 10-epc-up → provision → 20-ran-up(G0)
호스트 준비 → EPC 기동 → 가입자(UE1) → UAV UE attach까지 한 줄. SKIP_G0=1(G0 생략), FORCE_LOCAL/FORCE_EC2로 경로 강제. ⚠️ 진짜 Linux 셸(WSL/EC2)에서.

30-add-ue.sh <N> 멀티UE

bash scripts/30-add-ue.sh 2 # GCS bash scripts/30-add-ue.sh 3 # ROGUE
(eNB+UE) 쌍 N 추가(multi-eNB). 설정·compose 생성, 가입자 등록, attach + UE-to-UE 검증. N별 IP/enb_id/IMSI 자동.

40-g1-c2.sh G1 C2

bash scripts/40-g1-c2.sh
SITL(UAV-UE netns) 기동 → MAVLink UDP를 GCS-UE로. GCS(g1_gcs.py)가 다운링크 HEARTBEAT + 업링크 COMMAND_ACK 확인 = 양방향 C2.

50-g2b-gps.sh G2 GPS

bash scripts/50-g2b-gps.sh
uav-ue를 dahnet에 연결, SITL을 외부GPS로 재기동(serial2=GPS dahnet), gps_inject 가동 → fix_type=3 확인. GPS off-셀룰러.

g2_flight.py 폐루프

docker run -i --rm --network container:srsue_zmq2 \ dah-testbed-air python3 - < scripts/g2_flight.py
GCS가 셀룰러 C2로 GUIDED→arm→takeoff(10m)→land. 고도 추종 관측. G2-A 폐루프 검증.

udp_test.py 진단

docker cp scripts/udp_test.py <ue>:/tmp/ # rx: python3 /tmp/udp_test.py rx 12 # tx: python3 /tmp/udp_test.py tx <src> <dst>
UE-to-UE UDP(14550, MAVLink 전송 경로) 도달 점검 도구.

7. 통합 실행 가이드 (clone → 두 testbed)

이 repo(github.com/Kjhyun04/dah_testbed)에는 브로드캐스트 testbed(루트)와 셀룰러 testbed-4g(testbed-4g/)가 함께 들어 있다. git clone/git pull 한 번에 둘 다 받아지고/최신화된다. 단 testbed-4g는 브로드캐스트 testbed의 air 이미지 + dahnet 네트워크를 빌려쓰므로 실행 순서가 중요하다(브로드캐스트 → 4G).

  1. clone (둘 다 받기) — git clone https://github.com/Kjhyun04/dah_testbed · 이후 git pull 한 번이면 두 testbed 모두 최신화.
  2. 선행 — 브로드캐스트 testbed 빌드 — repo 루트에서 docker compose up -d --build. 이게 dah-testbed-air 이미지와 dah-testbed_dahnet 네트워크를 생성(= 4G의 SITL·GPS 채널 전제). 자원 아끼려면 이후 docker compose stop — 네트워크·이미지는 유지(컨테이너만 정지). 동시 가동은 자원경합만.
  3. 4G 환경 → G0 — cd testbed-4g && bash bootstrap.sh (WSL2 Ubuntu). EPC + UAV UE attach.
  4. 멀티UE — bash scripts/30-add-ue.sh 2(GCS) · 3(ROGUE).
  5. G1 C2-over-LTE — bash scripts/40-g1-c2.sh.
  6. G2 GPS off-셀룰러 + 폐루프 — bash scripts/50-g2b-gps.sh → g2_flight.py(비행).
어떤 "내부 기능"을 실행하나 — 요약.
• 브로드캐스트 testbed(루트): docker compose up -d --build (air 이미지·dahnet 제공이 핵심 역할). 자체 시나리오 제어는 gcs_cli.py.
• 셀룰러 testbed-4g: bootstrap.sh(G0) → 30-add-ue.sh(멀티UE) → 40-g1-c2.sh(C2) → 50-g2b-gps.sh+g2_flight.py(GPS+폐루프).
• 의존: testbed-4g는 dah-* 컨테이너는 안 쓰지만 air 이미지 + dahnet 네트워크는 필요 → 브로드캐스트 testbed를 최소 1회 빌드/up 해둘 것.
• git pull: 한 repo이므로 git pull 한 번에 두 testbed 모두 최신화.
